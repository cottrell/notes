Examples and reference implementations are becoming more useful. These might be called pseudo-frameworks.

Before agentic workflows, frameworks emerged despite high costs because disentangling a subsystem from its exemplar whole was frustrating and difficult for humans — the edges between content and framework are muddy, the surface area large, and examples create a lot of reading work to unpick. Now with agents, we have no concept of an agent being frustrated, and we sit somewhere in the space of agents being much better at this disentangling and humans largely discounting the cost of pointing an agent at someone else's plumbing and nudging it along until it maybe works.

From personal experience, the fact agents can persist memory basically effortlessly means that scatter human time can now be used to coax these disentangling and re-implementation works along. While before agents, a human would need huge swathes of uninterrupted time to concentrate and keep context in memory.

Agentic performance is largely equivariant to work interruptions while human performance is definitely not. This means it is no longer applies to oranges comparison where a human works for two hours and an agent works for five minutes at some cost. It is literally a human unable to do anything substantial on a such a problem given 12 ten minute work intervals over three days. But when you take even a poorly performing agent with 12 feedback iterations from the human you can actually accomplish a lot of this what is ultimately rather deterministic and "boring" but highly complex work.

## Other names

- Reference implementations
- Starter repos / starter kits / starter templates
- Blueprints (NVIDIA)
- Copy-paste templates / copy-this-repo patterns
- Scaffolds / minimal working examples
- AGENTS.md style files

## Sources and discussions

- [Microsoft Fabric — Operationalizing Agentic Applications](https://blog.fabric.microsoft.com) — "production-minded reference implementation" (Agentic Banking App) as the pattern; not a framework, just a concrete artifact to study and fork.
- [NVIDIA Build platform — Retail Agentic Commerce Blueprint](https://build.nvidia.com) — "Blueprints" positioned as reference implementations to adapt, not frameworks to hook into.
- Claude Code / Cursor communities (GitHub, LinkedIn, Instagram) — heavy sharing of "copy this repo" multi-agent team repos (e.g., `msitarzewski/agency-agents`). Explicit contrast with heavy frameworks.
- DeepLearning.AI / GitHub threads on Agentic RAG, LangGraph, Claude Agent SDK — "Not another framework — just a working example you can fork."
