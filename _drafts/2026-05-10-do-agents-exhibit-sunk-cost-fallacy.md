Consider some complicated agent session context and two scenario branches from that point:

A)

> QUESTION / THOUGHT X.
> Or is that too complicated?


B)
> QUESTION / THOUGHT X.
> <enter> ... agent starts thinking and working
> Or is that too complicated?


So in the latter (B) case the agent presumably has spent some budget thinking
and doing some work and then the human question of "Or is that too
complicated?" comes in.

This is sort of the setting where a sunk cost fallacy kind of behaviour might appear. Not sure we should call this a fallacy as that depends on the meta interpretation but wondering if this is studied on eval coherence or what it might even be called at the larger organizations.
